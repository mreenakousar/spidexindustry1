# Contributing

Thanks for contributing! Keep changes small and focused. Follow these guidelines:

- Branches: create feature branches from `main` named `feature/<short-desc>` or `fix/<short-desc>`.
- Commits: use clear messages (Imperative tense). Example: `Add orders table component`.
- Pull requests: provide a short description, screenshots if UI changes, and list of files changed.
- Code style: TypeScript, Next.js App Router conventions, and Tailwind for styles.
- Formatting: run `npm run build` locally to validate TypeScript/Next compilation before opening a PR.
- Tests: add unit or integration tests for new logic when applicable.

If you need to run the app locally:

```bash
npm install
npm run dev
```

For production build:

```bash
npm run build
npm start
```

Questions: open an issue or reach out to the maintainers.
